- `DualModePlayer`、`SimpleVideoPlayer`、`SLMAcquisition` 三个全局实例
- 视频流生成器按优先级尝试获取帧，可能导致混乱
- 场景切换时旧实例可能未完全清理

**影响**: 视频播放不稳定，切换场景后可能显示旧视频

**修复建议**:
```python
# 统一使用 DualModePlayer，废弃 SimpleVideoPlayer
# 在 init_player 中强制停止其他播放器
```

- 高并发时性能差

# 将导入移到函数外部
from core.slm.dual_mode_player import get_player
from core.slm.simple_video_player import get_video_player

async def video_stream_generator(channel: str, quality: int = 85):
    # 直接使用已导入的函数

#### 3. 场景切换时视频源不更新
**问题描述**:
- 切换场景后，视频流可能仍显示旧视频
- 需要刷新页面才能看到新视频

**影响**: 用户体验差

**修复建议**:
- 切换场景时强制停止并重新初始化播放器
- 添加视频源版本号，变化时强制刷新

### 中等问题

#### 4. 前端状态不一致
**问题描述**:
- `healthData` 和 `latestData.health` 分别维护
- 可能导致显示不一致

**位置**: `frontend/src/views/slm/Dashboard.vue`

#### 5. 模拟模式帧数同步
**问题描述**:
- 模拟模式下，前端计算的层数可能与视频实际帧数有偏差
- 100ms 轮询间隔可能导致跳层

**修复建议**:
- 使用更精确的帧数获取方式
- 从后端获取当前帧号而不是前端估算

#### 6. WebSocket 重连机制
**问题描述**:
- 断线后自动重连，但可能重连到旧状态
- 没有清理旧回调

## 建议优化

### 架构优化
1. **统一播放器管理**: 只保留 DualModePlayer，移除 SimpleVideoPlayer
2. **状态管理**: 使用 Pinia 或 Vuex 统一管理前端状态
3. **配置中心**: 将所有配置提取到单独文件

### 性能优化
1. **视频流优化**: 使用 WebRTC 或 WebSocket 传输视频，替代 MJPEG
2. **减少轮询**: 使用推送代替轮询获取状态
3. **懒加载**: 组件按需加载

### 代码质量
1. **类型检查**: 添加 Python 类型注解
2. **单元测试**: 添加关键功能测试
3. **日志规范**: 统一日志格式
