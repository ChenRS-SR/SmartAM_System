# SmartAM System 模拟模式使用指南

> 用于无硬件环境下的界面调试和功能测试

---

## 概述

模拟模式允许您在没有连接任何硬件（IDS相机、旁轴相机、红外热像仪、打印机）的情况下运行 SmartAM System，用于：

- 🖥️ **界面开发调试** - 无需硬件即可查看界面效果
- 🧪 **功能测试** - 测试数据采集流程和参数设置
- 📊 **演示展示** - 向他人展示系统功能
- 📝 **论文截图** - 生成演示图像用于论文

---

## 快速开始

### 方法一：一键启动（推荐）

双击运行 `start_with_simulation.bat`：

```bash
start_with_simulation.bat
```

这将自动：
1. 创建 `.env` 配置文件
2. 启用模拟模式
3. 启动后端服务

### 方法二：手动配置

1. **复制配置文件**：
   ```bash
   cd backend
   copy .env.example .env
   ```

2. **编辑 `.env` 文件**，添加模拟模式配置：
   ```ini
   SIMULATION_MODE=true
   SIMULATION_AUTO_FALLBACK=true
   ```

3. **正常启动**：
   ```bash
   python main.py
   ```

## 模拟数据说明

### IDS 相机（随轴视角）
- 模拟打印平台纹理
- 模拟打印头运动和发光效果
- 模拟已打印的层纹（随时间增长）
- 显示绿色 `[SIMULATION]` 标识

### 旁轴相机（整体视角）
- 模拟打印机框架结构
- 模拟打印件（立方体）
- 模拟打印头在运动
- 显示绿色 `[SIMULATION]` 标识

### 红外热像仪
- 模拟热床加热区域（底部高温区）
- 模拟熔池热点（随打印头移动）
- 模拟温度噪声和波动
- 温度范围：25°C（室温）~ 285°C（熔池）

### 打印机状态
- **位置**：模拟打印头在方形路径上运动
- **Z轴**：每秒自动上升 0.1mm（模拟打印过程）
- **温度**：热端 200±5°C，热床 60±2°C
- **进度**：每秒增加 0.1%
- **状态**：`Printing (Simulation)`

## 配置选项

### `.env` 文件配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `SIMULATION_MODE` | `false` | 强制启用模拟模式 |
| `SIMULATION_AUTO_FALLBACK` | `true` | 硬件连接失败时自动切换到模拟模式 |
| `IDS_ENABLE` | `true` | 是否尝试连接 IDS 相机 |
| `SIDE_CAMERA_ENABLE` | `true` | 是否尝试连接旁轴相机 |
| `FOTRIC_ENABLE` | `true` | 是否尝试连接红外相机 |

### 代码中启用

在 Python 代码中创建 DAQ 实例时启用：

```python
from core.data_acquisition import DataAcquisition, AcquisitionConfig

config = AcquisitionConfig(
    simulation_mode=True,  # 强制模拟模式
    simulation_auto_fallback=True  # 允许自动回退
)

daq = DataAcquisition(config)
```

## 使用场景

### 场景 1：纯界面开发
只需要查看界面效果，无需真实数据：

```ini
# .env
SIMULATION_MODE=true
```

### 场景 2：混合模式
尝试连接硬件，失败时使用模拟数据：

```ini
# .env
SIMULATION_MODE=false
SIMULATION_AUTO_FALLBACK=true
IDS_ENABLE=true
SIDE_CAMERA_ENABLE=true
FOTRIC_ENABLE=true
```

### 场景 3：部分硬件
已连接部分硬件，其余使用模拟：

```ini
# .env
SIMULATION_MODE=false
SIMULATION_AUTO_FALLBACK=true
IDS_ENABLE=true       # 连接真实 IDS
SIDE_CAMERA_ENABLE=false  # 使用模拟旁轴相机
FOTRIC_ENABLE=false   # 使用模拟红外相机
```

## 注意事项

1. **模拟数据不保存到文件** - 当前实现中，模拟数据的图像不会保存到硬盘（仅用于显示）

2. **PacNet 推理** - 如果使用 PacNet 模型，模拟图像也能进行推理（但结果可能无实际意义）

3. **打印机控制** - 模拟模式下发送到打印机的 G-code 命令会被记录但不会实际执行

4. **性能** - 模拟模式下的图像生成会消耗一定的 CPU 资源

## 故障排除

### 问题：模拟模式没有启用
**检查**：
- `.env` 文件是否存在
- `SIMULATION_MODE=true` 是否设置
- 重启服务后配置才生效

### 问题：模拟图像卡顿
**解决**：
- 这是正常的，因为图像需要实时生成
- 降低视频流分辨率或减少帧率

### 问题：前端无法连接
**检查**：
- 后端服务是否启动成功
- 浏览器控制台是否有跨域错误
- 防火墙是否阻止了端口 8000

---

## 从模拟模式切换到真实硬件

1. 连接所有硬件设备
2. 修改 `.env`：
   ```ini
   SIMULATION_MODE=false
   SIMULATION_AUTO_FALLBACK=true
   ```
3. 重启服务
4. 检查日志确认真实设备已连接

---

**提示**：模拟模式主要用于开发和调试，正式采集数据时请务必使用真实硬件！
