# SmartAM System 模拟模式使用指南

> 用于无硬件环境下的界面调试和功能测试

---

## 概述

模拟模式允许您在没有连接任何硬件（IDS相机、旁轴相机、红外热像仪、打印机）的情况下运行 SmartAM System，用于：

- 🖥️ **界面开发调试** - 无需硬件即可查看界面效果
- 🧪 **功能测试** - 测试数据采集流程和参数设置
- 📊 **演示展示** - 向他人展示系统功能
- 📝 **论文截图** - 生成演示图像用于论文

---

## 快速开始

### 方法一：一键启动（推荐）

#### 完整模拟模式（相机+打印机）
```bash
start_simulation_full.bat
```

#### 仅相机模拟（需要真实 OctoPrint）
```bash
start_with_simulation.bat
```

### 方法二：手动配置

1. **复制配置文件**：
   ```bash
   cd backend
   copy .env.example .env
   ```

2. **编辑 `.env` 文件**，添加模拟模式配置：
   ```ini
   # 完整模拟（相机+打印机）
   SIMULATION_MODE=true
   SIMULATION_AUTO_FALLBACK=true
   OCTOPRINT_SIMULATION=true
   OCTOPRINT_SIMULATION_AUTO_FALLBACK=true
   
   # 或仅 OctoPrint 模拟（有真实相机，无打印机）
   SIMULATION_MODE=false
   OCTOPRINT_SIMULATION=true
   ```

3. **正常启动**：
   ```bash
   python main.py
   ```

## 模拟模式类型

### 1. 完整模拟模式（SIMULATION_MODE=true）

**适用场景**：无任何硬件，纯界面调试

**模拟内容**：
- ✅ IDS 相机（随轴视角）
- ✅ 旁轴相机（整体视角）
- ✅ 红外热像仪
- ✅ OctoPrint API（打印机状态/温度/位置）
- ✅ G-code 命令响应

**启动方式**：
```bash
start_simulation_full.bat
```

### 2. 仅 OctoPrint 模拟（OCTOPRINT_SIMULATION=true）

**适用场景**：有真实相机，但无打印机/OctoPrint

**模拟内容**：
- ✅ 真实 IDS/旁轴/热像相机
- ✅ OctoPrint API（打印机状态/温度/位置）
- ✅ G-code 命令响应

**配置方式**：
```ini
# .env
SIMULATION_MODE=false
OCTOPRINT_SIMULATION=true
IDS_ENABLE=true
SIDE_CAMERA_ENABLE=true
FOTRIC_ENABLE=true
```

### 3. 自动回退模式（AUTO_FALLBACK）

**适用场景**：优先使用真实硬件，失败时自动模拟

**配置方式**：
```ini
# .env
SIMULATION_MODE=false
SIMULATION_AUTO_FALLBACK=true
OCTOPRINT_SIMULATION=false
OCTOPRINT_SIMULATION_AUTO_FALLBACK=true
```

## 模拟数据说明

### IDS 相机（随轴视角）
- 模拟打印平台纹理
- 模拟打印头运动和发光效果
- 模拟已打印的层纹（随时间增长）
- 显示绿色 `[SIMULATION]` 标识

### 旁轴相机（整体视角）
- 模拟打印机框架结构
- 模拟打印件（立方体）
- 模拟打印头在运动
- 显示绿色 `[SIMULATION]` 标识

### 红外热像仪
- 模拟热床加热区域（底部高温区）
- 模拟熔池热点（随打印头移动）
- 模拟温度噪声和波动
- 温度范围：25°C（室温）~ 285°C（熔池）

### OctoPrint 模拟

| 数据类型 | 模拟内容 |
|----------|----------|
| **打印机状态** | Operational/Printing/Paused |
| **温度** | 热端 200±2°C，热床 60±2°C |
| **位置** | X/Y 方形运动，Z 每秒+0.05mm |
| **进度** | 每秒+0.1%，模拟1小时打印 |
| **G-code** | 记录命令，更新模拟状态 |

### 模拟 G-code 响应

| 命令 | 模拟行为 |
|------|----------|
| `M104 Sxxx` | 设置热端目标温度 |
| `M140 Sxxx` | 设置热床目标温度 |
| `M220 Sxxx` | 设置速度倍率 |
| `M221 Sxxx` | 设置流量倍率 |
| `M290 Zxxx` | 调整 Z 偏移 |
| `M114` | 返回当前位置 |
| `M105` | 返回当前温度 |

## 配置选项

### `.env` 文件配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `SIMULATION_MODE` | `false` | 强制启用完整模拟模式 |
| `SIMULATION_AUTO_FALLBACK` | `true` | 硬件连接失败时自动切换到模拟模式 |
| `OCTOPRINT_SIMULATION` | `false` | 仅启用 OctoPrint 模拟 |
| `OCTOPRINT_SIMULATION_AUTO_FALLBACK` | `true` | OctoPrint 连接失败时自动模拟 |
| `IDS_ENABLE` | `true` | 是否尝试连接 IDS 相机 |
| `SIDE_CAMERA_ENABLE` | `true` | 是否尝试连接旁轴相机 |
| `FOTRIC_ENABLE` | `true` | 是否尝试连接红外相机 |
| `SIMULATION_HOTEND_TEMP` | `200` | 模拟热端温度 |
| `SIMULATION_BED_TEMP` | `60` | 模拟热床温度 |

### 代码中启用

```python
from core.data_acquisition import DataAcquisition, AcquisitionConfig

config = AcquisitionConfig(
    simulation_mode=True,              # 完整模拟
    simulation_auto_fallback=True,
    octoprint_simulation=True,         # 仅 OctoPrint 模拟
    octoprint_simulation_auto_fallback=True
)

daq = DataAcquisition(config)
```

```ini
# .env
```

## 故障排除

### 问题：模拟模式没有启用
**检查**：
- `.env` 文件是否存在
- 配置项是否正确设置（`true` 小写）
- 重启服务后配置才生效

### 问题：OctoPrint 模拟不工作
**检查**：
- 查看日志是否有 `[OctoPrint模拟] 已启用`
- 检查 API 返回是否包含 `"simulation": true`

### 问题：真实 OctoPrint 无法连接
**检查**：
- OctoPrint 是否运行
- API Key 是否正确
- `.env` 中的 `OCTOPRINT_URL` 是否正确

### 问题：模拟图像卡顿
**解决**：
- 这是正常的，因为图像需要实时生成
- 降低视频流分辨率或减少帧率
