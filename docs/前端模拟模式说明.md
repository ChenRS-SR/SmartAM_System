# 前端模拟模式说明

> 无需后端服务，纯前端运行，使用模拟数据

---

## 快速开始

### 一键启动（推荐）

```bash
start_frontend_only.bat
```

然后访问：**http://localhost:5173**

### 手动启动

```bash
cd frontend

# 启用模拟模式
echo VITE_MOCK_MODE=true > .env.local

# 安装依赖（如果还没装）
npm install

# 启动
npm run dev
```

- 实时更新

- 随机波动

## 技术实现

### 文件结构
```
frontend/src/
├── utils/
│   ├── mockData.js        # 模拟数据生成器
│   └── api.js             # 带模拟回退的 API
├── composables/
│   └── useVideoStream.js  # 视频流模拟
└── ...
```

### 工作原理

1. **API 层拦截**
   ```javascript
   // 尝试连接真实后端
   try {
     return await axios.get('/api/xxx')
   } catch (e) {
     // 失败则返回模拟数据
     return mockData
   }
   ```

2. **视频流模拟**
   ```javascript
   // 使用 Canvas 生成动画帧
   const canvas = document.createElement('canvas')
   // 绘制模拟画面...
   return canvas.toDataURL('image/jpeg')
   ```

3. **WebSocket 模拟**
   ```javascript
   // 定时发送模拟数据
   setInterval(() => {
     onMessage(mockWebSocketData)
   }, 1000)
   ```

## 常见问题

### Q: 视频流是静态图片吗？
**A:** 不是，是 Canvas 实时生成的动画，能看到打印头在移动。

### Q: 可以保存数据吗？
**A:** 纯前端模式不能保存（无后端），如需保存请使用完整模拟模式。

### Q: 如何添加新的模拟数据？
**A:** 修改 `frontend/src/utils/mockData.js`，添加新的数据生成函数。

### Q: 模拟数据可以自定义吗？
**A:** 可以，修改 mockData.js 中的参数，如温度范围、运动速度等。

### Q: 为什么界面显示 "Simulation"？
**A:** 这是特意添加的标识，提醒当前是模拟模式，避免与真实数据混淆。

## 配置说明

### 环境变量

| 变量 | 值 | 说明 |
|------|-----|------|
| `VITE_MOCK_MODE` | `true` | 强制使用模拟模式 |
| `VITE_MOCK_MODE` | `false` | 尝试连接真实后端 |
| `VITE_API_BASE_URL` | `http://localhost:8000` | 后端地址 |

### 配置优先级

1. `.env.local`（最高优先级，不会被 git 提交）
2. `.env.development`（开发环境默认）
3. `.env`（通用默认）
