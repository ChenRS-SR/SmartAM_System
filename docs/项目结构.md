## 三、设计决策说明

### 3.1 SLS设备驱动选型

**Fotric 628CH - HTTP API vs SDK**

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| SDK DLL | 功能完整 | 依赖Windows、版本兼容性差 | ❌ |
| HTTP API | 跨平台、简单易用、无依赖 | 功能较简单 | ✅ |

**决策**: 使用HTTP API，通过 `192.168.1.100:10080` 访问

**振动传感器 - Modbus RTU**

- 协议: Modbus RTU over RS485
- 波特率: 9600
- 数据位: 8，停止位: 1，无校验
- 寄存器: 0x3A-0x43 (速度/位移/频率/温度)

### 3.2 模型文件存放

**问题**：原来引用外部项目路径的模型，移植时会丢失  
**解决**：
- 将 `model_full.pt` 和 `scaler.pkl` 复制到 `backend/weights/`
- 代码优先查找项目内路径，回退到外部路径（兼容旧配置）

```python
# 优先级：
# 1. SmartAM_System/backend/weights/model_full.pt
# 2. pacfnet_project/saved_models/full/model_full.pt
```

**移植时只需复制整个项目，无需外部依赖**

### 3.3 API 模块化

| 模块 | 功能 | 接口示例 |
|------|------|----------|
| `printer.py` | FDM打印机控制 | `POST /api/printer/connect` |
| `slm.py` | SLM设备控制 | `POST /api/slm/start` |
| `sls.py` | SLS设备控制 | `POST /api/sls/start` |
| `camera.py` | 相机控制 | `GET /api/camera/status` |
| `inference.py` | 模型推理 | `GET /api/inference/prediction` |
| `control.py` | 调控策略 | `POST /api/control/pid_params` |
| `data.py` | 数据管理 | `POST /api/data/record/start` |

## 四、移植指南

### 4.1 复制项目到新环境

```bash
# 1. 复制整个 SmartAM_System 目录
scp -r SmartAM_System user@new_server:/path/to/

# 2. 可选：同时复制模型文件（如果 weights/ 目录为空）
scp pacfnet_project/saved_models/full/model_full.pt new_server:/path/to/SmartAM_System/backend/weights/
scp pacfnet_project/saved_models/full/scaler.pkl new_server:/path/to/SmartAM_System/backend/weights/
```

### 4.2 安装依赖

```bash
cd SmartAM_System/backend
pip install -r requirements.txt
```

### 4.3 配置（可选）

创建 `.env` 文件：
```
# Fotric 热像仪配置 (SLS)
FOTRIC_IP=192.168.1.100
FOTRIC_PORT=10080

# 振动传感器配置 (SLS/SLM)
VIBRATION_COM=COM5
VIBRATION_BAUD=9600

# 舵机配置 (SLS)
SERVO_COM=COM6

# OctoPrint 配置 (FDM)
OCTOPRINT_URL=http://127.0.0.1:5000
OCTOPRINT_API_KEY=your_api_key

# 模型路径（可选，默认使用 weights/ 目录）
MODEL_PATH=weights/model_full.pt
```

### 4.4 启动

```bash
python main.py
```

**即使缺少外部依赖，服务也能启动**（使用模拟模式）

## 六、常见问题

### Q1: weights/ 目录中的模型文件太大，git 提交不了？

**方案 A（推荐）**：使用 Git LFS
```bash
git lfs track "backend/weights/*.pt"
git lfs track "backend/weights/*.pkl"
```

**方案 B**：不提交模型文件，部署时手动复制
```bash
# .gitignore
backend/weights/*.pt
backend/weights/*.pkl
```

### Q2: SLS设备无法连接？

**检查清单：**
1. Fotric热像仪IP是否可达 (`ping 192.168.1.100`)
2. 振动传感器COM口是否正确
3. 舵机COM口是否正确
4. 是否被其他程序占用

**使用模拟模式测试：**
```python
sls = get_sls_acquisition(simulation_mode=True)
```

### Q3: 没有 GPU，如何运行？

修改配置：
```python
# 在 pacnet_inference.py 中
device = "cpu"  # 自动 fallback
```
