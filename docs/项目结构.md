## 二、设计决策说明

### 2.1 模型文件存放

**问题**：原来引用外部项目路径的模型，移植时会丢失  
**解决**：
- 将 `model_full.pt` 和 `scaler.pkl` 复制到 `backend/weights/`
- 代码优先查找项目内路径，回退到外部路径（兼容旧配置）

```python
# 优先级：
# 1. SmartAM_System/backend/weights/model_full.pt
# 2. pacnet_project/saved_models/full/model_full.pt
```

**移植时只需复制整个项目，无需外部依赖**

---

### 2.2 外部模块依赖

**问题**：`fotric_driver` 和 `coordinator` 依赖 `pacnet_project`  
**解决**：创建 **硬件适配层** (`hardware_adapter.py`)

```
策略：适配器模式（Adapter Pattern）

┌─────────────────────────────────────────────────────────┐
│  SmartAM_System                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  hardware_adapter.py (适配层)                    │   │
│  │  ┌───────────────┐    ┌───────────────────────┐│   │
│  │  │ 尝试导入外部   │───→│ 成功：使用外部模块    ││   │
│  │  │ pacnet_project│    │                       ││   │
│  │  └───────────────┘    └───────────────────────┘│   │
│  │          │                     ↑               │   │
│  │          ↓ 失败               │               │   │
│  │  ┌───────────────┐    ┌───────────────────────┐│   │
│  │  │ 使用简化版    │───→│ 模拟模式 / API 回退   ││   │
│  │  │ 实现          │    │                       ││   │
│  │  └───────────────┘    └───────────────────────┘│   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**好处**：
1. **可独立运行**：即使外部项目不存在，也能启动（使用模拟数据）
2. **平滑降级**：外部模块失败时自动切换到备用方案
3. **便于移植**：不强制依赖特定目录结构

---

### 2.3 API 模块化

**问题**：最初只有一个 `printer.py`，功能混杂  
**解决**：按功能拆分为 5 个模块

| 模块 | 功能 | 接口示例 |
|------|------|----------|
| `printer.py` | 打印机控制 | `POST /api/printer/connect` |
| `camera.py` | 相机控制 | `GET /api/camera/status` |
| `inference.py` | 模型推理 | `GET /api/inference/prediction` |
| `control.py` | 调控策略 | `POST /api/control/pid_params` |
| `data.py` | 数据管理 | `POST /api/data/record/start` |

**好处**：
1. **职责清晰**：每个模块只负责一类功能
2. **便于扩展**：新增功能只需添加模块
3. **团队协作**：不同开发者可同时修改不同模块

## 三、移植指南

### 3.1 复制项目到新环境

```bash
# 1. 复制整个 SmartAM_System 目录
scp -r SmartAM_System user@new_server:/path/to/

# 2. 可选：同时复制模型文件（如果 weights/ 目录为空）
scp pacnet_project/saved_models/full/model_full.pt new_server:/path/to/SmartAM_System/backend/weights/
scp pacnet_project/saved_models/full/scaler.pkl new_server:/path/to/SmartAM_System/backend/weights/
```

### 3.2 安装依赖

```bash
cd SmartAM_System/backend
pip install -r requirements.txt
```

### 3.3 配置（可选）

创建 `.env` 文件：
```
# 相机配置
FOTRIC_IP=192.168.1.100
FOTRIC_PORT=10080

# OctoPrint 配置
OCTOPRINT_URL=http://127.0.0.1:5000
OCTOPRINT_API_KEY=your_api_key

# 模型路径（可选，默认使用 weights/ 目录）
MODEL_PATH=weights/model_full.pt
```

### 3.4 启动

```bash
python main.py
```

**即使缺少外部依赖，服务也能启动**（使用模拟模式）

## 五、常见问题

### Q1: weights/ 目录中的模型文件太大，git 提交不了？

**方案 A（推荐）**：使用 Git LFS
```bash
git lfs track "backend/weights/*.pt"
git lfs track "backend/weights/*.pkl"
```

**方案 B**：不提交模型文件，部署时手动复制
```bash
# .gitignore
backend/weights/*.pt
backend/weights/*.pkl
```

### Q2: 外部项目更新了，如何同步？

**fotric_driver 更新**：
- 如果接口不变：无需修改（适配器自动使用新版）
- 如果接口变化：更新 `hardware_adapter.py` 中的适配代码

**模型更新**：
- 复制新模型到 `weights/` 目录
- 重启服务即可

### Q3: 没有 GPU，如何运行？

修改配置：
```python
# 在 data_acquisition.py 中
daq = DataAcquisitionSystem(config)
daq.initialize()

# 或者在 pacnet_inference.py 中指定
device = "cpu"  # 自动 fallback
```
